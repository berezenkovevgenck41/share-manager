# SDR 0001: Вибір базового технологічного стеку

## Контекст
Необхідно розробити клієнтський вебзастосунок "Менеджер шерингу речей", який повністю відповідає обмеженням статичного хостингу GitHub Pages (відсутність серверної частини бекенду та бази даних). 

## Рішення
Обрано наступний стек технологій:
- **UI/UX:** HTML5 та CSS3. Використано нативні модулі CSS Grid та Flexbox для забезпечення адаптивного макета. Відмова від важких UI-фреймворків (Bootstrap, Tailwind) задля мінімізації розміру проєкту та спрощення структури.
- **Логіка застосунку:** Vanilla JavaScript (ES6). Відмова від використання фронтенд-фреймворків (React, Vue, Angular), оскільки складність застосунку не потребує віртуального DOM, а всі маніпуляції зі списком речей ефективно виконуються нативними методами JavaScript.

## Наслідки
**Позитивні:**
- Застосунок є повністю статичним, що ідеально підходить для деплою на GitHub Pages.
- Максимальна швидкість завантаження через відсутність зовнішніх залежностей.
- Легкість розуміння коду під час тестування напарником.

**Негативні:**
- Відсутність компонентного підходу може ускладнити масштабування, якщо застосунок у майбутньому виросте до десятків сторінок.
