# SDR 0002: Вибір методу локального збереження даних

## Контекст
Оскільки архітектура застосунку виключає наявність серверної бази даних (відповідно до вимог розгортання на GitHub Pages), необхідно обрати механізм збереження записів про позичені речі безпосередньо у браузері користувача.

## Рішення
Для збереження даних обрано **Web Storage API (localStorage)**.
Альтернативний варіант (IndexedDB) був відхилений через надмірну складність API для поточної структури даних (простий масив об'єктів).

## Наслідки
**Позитивні:**
- Дані не втрачаються після закриття вкладки або оновлення сторінки.
- Дані зберігаються у форматі JSON, що дозволяє легко їх парсити та оновлювати у вигляді масивів об'єктів JavaScript.
- Метод працює синхронно, що значно спрощує код.

**Негативні / Обмеження:**
- Жорстка прив'язка даних до конкретного браузера та пристрою. Синхронізація (наприклад, між смартфоном та ПК користувача) неможлива.
- Дані будуть безповоротно втрачені, якщо користувач очистить кеш браузера або використає режим Інкогніто.
